The path for SREs, ops engineers, and cluster administrators who keep Kubernetes running in production. After Fundamentals, this course focuses on operating a secure and governable multi-team cluster.

**What you'll learn:**

- **Cluster access and identity**: kubeconfig, API access control, ServiceAccounts, and RBAC
- **Workload and tenant security**: cloud-native security model, Pod Security Standards, Linux security contexts
- **Governance guardrails**: ResourceQuotas and LimitRanges for multi-tenant teams
- **Storage provisioning**: StorageClass fundamentals for dynamic provisioning
- **Production operations**: logging, observability, and certificate management

**Who is this for?**

SREs, system administrators, platform teams, and anyone preparing for the **CKA** (Certified Kubernetes Administrator) certification.

**Prerequisites:** Kubernetes Fundamentals (or equivalent knowledge of kubectl, objects, and namespaces).

**Course structure:**

13 chapters across 5 modules, progressing from cluster access and security controls to governance and production operations. Every chapter includes YAML examples, kubectl commands, and quizzes.

Recommended next step: **Kubernetes Platform Engineer** (to extend Kubernetes with CRDs and operators).
